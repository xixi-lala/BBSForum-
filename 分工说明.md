# BBS技术社区 - 项目分工说明

## 项目概述

BBS技术社区是一个基于 Jakarta Servlet 5.0 + JSP + JSTL 的论坛系统，使用嵌入式 Tomcat 10.1 启动，连接 MySQL 8.0 数据库。团队共5人，每人全栈开发（前端页面 + 后端接口 + 数据库建表）。

---

## 分工明细

### 组长：帖子核心功能 + 公共架构 + 创新功能（工作量 27%）

**数据库建表**：posts、replies、user_follows、post_likes、post_favorites

**后端接口**：
- `HomeServlet.java` — 首页帖子列表
- `PostServlet.java` — 帖子CRUD、回复、AI总结、积分操作(发帖+10/回复+2)
- `InteractionServlet.java` — 关注/点赞/收藏 + 积分(点赞+3)
- `CoverServlet.java` — SVG封面图生成

**前端页面**：
- `layouts/main.jsp` — 全局布局（导航栏/侧边栏/底栏/右侧面板/统一弹窗）
- `post/list.jsp` — 帖子列表
- `post/create_content.jsp` — 发帖页面（Markdown编辑器+实时预览）
- `post/detail_content.jsp` — 帖子详情页（含回复/关注/点赞/收藏）

**公共架构**：
- `Main.java` — 嵌入式Tomcat启动入口
- `DBUtil.java` — 数据库连接
- `AiUtil.java` — AI接口调用
- `ContentUtil.java` — 内容处理
- `PostMapper.java` — 帖子结果集映射
- `StatsFilter.java` — 实时数据统计
- `PasswordUtil.java` — 密码加密
- `EncodingFilter.java` — 编码过滤
- `AuthFilter.java` — 登录认证
- `js/common.js` — 公共JavaScript
- `css/global.css` — 全局样式
- `pom.xml` — 项目依赖
- `database/init.sql` — 合并汇总

### 组员A：置顶 + 加精 + 搜索 + 热度榜 + 二级确认（工作量 17%）

**数据库**：posts 表扩展字段（is_top置顶、is_elite加精）

**后端接口**：
- `AdminPostServlet.java` — 置顶（板块置顶/全局置顶）、加精、管理员帖子管理列表
- `PostServlet.java`（搜索部分） — 帖子搜索（LIKE匹配+分页）、搜索结果逻辑
- `HotServlet.java` — 热度榜（按浏览量排序）

**前端页面**：
- `admin/post_manage.jsp` — 帖子管理页面
- `admin/post_manage_content.jsp` — 管理内容页面
- `layouts/main.jsp`（搜索框组件）
- `post/list.jsp`（搜索结果列表，复用首页模板）
- `post/hot_content.jsp` — 热度榜页面

**管理入口**：顶部导航栏用户名旁 → 管理
**搜索入口**：顶部导航栏搜索框

### 组员B：用户系统 + 签到 + 登录奖励（工作量 19%）

**数据库建表**：users、daily_checkins

**后端接口**：
- `UserServlet.java` — 注册、登录、退出、登录+2积分
- `UserProfileServlet.java` — 个人资料查询/编辑、密码修改、每日签到

**前端页面**：
- `user/login.jsp` → `user/login_content.jsp` — 登录页
- `user/register.jsp` → `user/register_content.jsp` — 注册页
- `user/profile.jsp` → `user/profile_content.jsp` — 个人中心（积分展示）
- `user/profile_edit.jsp` → `user/profile_edit_content.jsp` — 资料编辑

---

### 组员C：板块管理 + 文章编辑 + 分板块展示（工作量 19%）

**数据库建表**：categories

**后端接口**：
- `CategoryServlet.java` — 按板块筛选帖子、板块帖子列表
- `AdminCategoryServlet.java` — 板块CRUD（添加/编辑/删除）

**前端页面**：
- `post/edit_content.jsp` — 帖子编辑页
- `admin/categories.jsp` → `admin/categories_content.jsp` — 板块管理页
- `admin/category_edit.jsp` → `admin/category_edit_content.jsp` — 编辑板块页

**积分协作**：需求详情页回复列表"采纳"交互

---

### 组员D：需求悬赏 + 积分流转 + 排行榜（工作量 18%）

**数据库建表**：demands、demand_replies、score_logs

**后端接口**：
- `DemandServlet.java` — 需求发布/列表/详情/编辑、需求回复、采纳回复（积分事务）
- `ScoreServlet.java` — 积分排行榜、积分流水记录

**前端页面**：
- `demand/detail.jsp` → `demand/detail_content.jsp` — 需求详情（回复+采纳弹窗）
- `post/demand_content.jsp` — 需求列表
- `post/demand_create_content.jsp` — 发布需求
- `post/demand_edit_content.jsp` — 编辑需求
- `score/rank.jsp` → `score/rank_content.jsp` — 排行榜
- `score/record.jsp` → `score/record_content.jsp` — 积分流水

## 开发依赖关系

```
组员C（板块表）
    |
    +--- 组长（帖子核心，发帖要选板块）
    |       |
    |       +--- 组员A（置顶加精，依赖 posts.is_top/is_elite 字段）
    |
    +--- 组员D（需求独立表，不需要依赖其他模块）
    |
    +--- 组员B（用户系统，贯穿全程，注册登录是入口）
```

- 组员C先完成板块表 → 组长才能做发帖功能
- 组长完成帖子表 → 组员A才能做置顶加精
- 组员B的用户系统全程需要，先完成
- 组员D的需求悬赏独立建表，可并行开发

**详细分工方案请参考**：`BBS论坛项目分工方案.md`
