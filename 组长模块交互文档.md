# BBS论坛系统 - 组长模块交互文档

> 版本：v2.0（优化版） | 更新日期：2026-06-02

---

## 一、模块概览

组长负责以下 3 个核心功能模块：

| 功能 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` `/index` `/home` | 帖子列表 + 板块导航 + 分页 |
| 发布帖子 | `/post/create` | 发帖表单 + Markdown 编辑 + 图片上传 |
| 帖子详情 | `/post/detail?id=` | 帖子内容 + 回复列表 + AI 总结 |

---

## 二、路由与接口

### 2.1 首页 — 帖子列表

**`GET /` 或 `GET /index` 或 `GET /home`**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码，从 1 开始 |
| `refresh` | int | 否 | 0 | 设为 1 强制刷新板块缓存 |

**返回属性：**

```
request.setAttribute("postList")       → List<Map>  帖子列表（每页 20 条）
request.setAttribute("currentPage")    → int        当前页码
request.setAttribute("totalPages")     → int        总页数
request.setAttribute("totalPosts")     → int        帖子总数
request.setAttribute("pageSize")       → int        每页条数

applicationScope:
  categoryList → List<Map>  板块列表 [{id, name, description}]
```

**帖子 Map 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 帖子ID |
| `title` | String | 标题 |
| `summary` | String | 内容摘要（前 120 字） |
| `imageUrl` | String | 封面图 URL |
| `isTop` | int | 0=普通 1=板块置顶 2=全局置顶 |
| `isElite` | int | 0=否 1=精华 |
| `viewCount` | int | 浏览量 |
| `createdAt` | String | 创建时间 |
| `authorName` | String | 作者用户名 |
| `categoryName` | String | 所属板块名 |

### 2.2 板块过滤

**`GET /category?id={categoryId}&page={page}`**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | int | 是 | 板块ID |
| `page` | int | 否 | 页码 |

### 2.3 发布帖子

**`GET /post/create`** — 显示发帖表单（需登录）

- 未登录 → 重定向到 `/user/login`
- 已登录 → 转发到 `/post/create.jsp`

**`POST /post/create`** — 提交帖子（需登录）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | String | 是 | 标题，最长 100 字 |
| `content` | String | 是 | 内容，支持 Markdown |
| `categoryId` | int | 是 | 板块ID |
| `keywords` | String | 否 | 关键词，逗号分隔，最长 200 字 |
| `coverImage` | file | 否 | 上传封面图，最大 5MB |
| `imageUrl` | String | 否 | 封面图 URL（上传优先） |

- 成功 → 重定向到 `/post/detail?id={新帖ID}`
- 失败 → 返回表单页并显示错误信息

**`POST /post/uploadImage`** — 上传内联图片（需登录）

| 参数 | 类型 | 说明 |
|------|------|------|
| `coverImage` | file | 图片文件，支持 jpg/png/gif/webp |

返回 JSON：
```json
{"markdown": "![图片](/BBSForum/uploads/xxx.jpg)", "url": "/BBSForum/uploads/xxx.jpg"}
```

### 2.4 帖子详情

**`GET /post/detail?id={postId}`**（无需登录）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | int | 是 | 帖子ID |

**返回属性：**

```
request.setAttribute("post")           → Map        帖子详情
request.setAttribute("replyList")      → List<Map>  回复列表
request.setAttribute("replyCount")     → int        回复数量
request.setAttribute("relatedPosts")   → List<Map>  相关推荐（最多5篇）
```

**帖子详情 Map 字段（比列表多以下字段）：**

| 字段 | 说明 |
|------|------|
| `content` | 原始 Markdown 内容 |
| `contentRendered` | 渲染后的 HTML |
| `keywords` | 原始关键词字符串 |
| `aiSummary` | AI 总结文本（可能为空） |
| `userId` | 作者ID |
| `categoryId` | 板块ID |

### 2.5 发表回复

**`POST /post/reply`**（需登录）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `content` | String | 是 | 回复内容 |
| `postId` | int | 是 | 所属帖子ID |

- 成功/失败 → 重定向回 `/post/detail?id={postId}`

### 2.6 AI 总结

**`POST /post/aiSummary`**（需登录）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | int | 是 | 帖子ID |

返回 JSON：
```json
{"summary": "本文介绍了 Java 21 虚拟线程的核心特性..."}
// 或 {"error": "AI接口调用失败，请重试"}
```

### 2.7 编辑帖子

**`GET /post/edit?id={postId}`**（需作者或管理员）
**`POST /post/edit`**（需作者或管理员）

参数同发帖，但 `id` 为已有帖子ID。

### 2.8 删除帖子

**`GET /post/delete?id={postId}`**（需作者或管理员）

- 成功 → 重定向到首页
- 无权限 → 重定向到帖子详情页

## 三、文件结构

```
src/main/java/com/bbs/
├── controller/
│   ├── HomeServlet.java        # 首页 + 板块缓存（全局共享）
│   ├── CategoryServlet.java    # 板块过滤 + 分页
│   └── PostServlet.java        # 发帖/详情/回复/编辑/删除/AI/图片
├── util/
│   ├── DBUtil.java             # 数据库连接
│   ├── ContentUtil.java        # Markdown 渲染 + 摘要
│   ├── AiUtil.java             # AI 总结 API
│   └── PostMapper.java         # ResultSet → Map 映射 ★新增

src/main/webapp/
├── index.jsp                   # 入口（转发到 /index）
├── WEB-INF/home.jsp            # 首页模板
├── layouts/main.jsp            # 全局布局（导航+侧边栏+内容区）
├── post/
│   ├── list.jsp                # 帖子列表卡片 + 分页导航
│   ├── create.jsp              # 发帖模板
│   ├── create_content.jsp      # 发帖表单
│   ├── detail.jsp              # 详情模板
│   ├── detail_content.jsp      # 详情+回复+相关推荐
│   ├── edit.jsp                # 编辑模板
│   └── edit_content.jsp        # 编辑表单
└── category/list.jsp           # 板块页面模板
```

## 四、数据流

```
浏览器请求 / 或 /index
    │
    ▼
HomeServlet.doGet()
    │
    ├── 1. 检查 ServletContext.categoryList
    │       └── 为 null → 从 DB 加载板块列表 → 存入 ServletContext
    │       └── ?refresh=1 → 强制刷新
    │
    ├── 2. 解析 ?page= 参数
    │
    ├── 3. SELECT COUNT(*) FROM posts        ← 统计总数
    │
    ├── 4. SELECT ... LIMIT 20 OFFSET ?      ← 分页查询
    │       └── PostMapper.mapPostRow(rs)    ← 统一映射
    │
    ├── 5. request.setAttribute(...)          ← 传递数据
    │
    └── 6. forward → /WEB-INF/home.jsp
            └── include main.jsp → include post/list.jsp
```

## 五、v2.0 优化内容

| 优化项 | 说明 |
|--------|------|
| **分页** | 首页和板块页支持 `?page=` 参数，每页 20 条，底部显示页码导航 |
| **参数校验** | `CategoryServlet` 增加 `NumberFormatException` 处理 |
| **公共映射** | 新增 `PostMapper` 工具类，消除 ResultSet 映射重复代码 |
| **日志** | 所有 `printStackTrace()` 替换为 `java.util.logging.Logger`，记录中文日志 |
| **板块缓存** | 从 `session` 改为 `ServletContext` 全局共享，支持 `?refresh=1` 刷新 |
| **连接复用** | AI 总结方法复用同一个数据库连接，减少开销 |
| **JSP 同步** | 所有 JSP 中 `categoryList` 统一从 `applicationScope` 读取 |

## 六、数据库表

**涉及的表：**

- `categories` — 板块表（id, name, description, sort_order）
- `posts` — 帖子表（id, title, content, image_url, keywords, ai_summary, is_top, is_elite, view_count, user_id, category_id, created_at, updated_at）
- `replies` — 回复表（id, content, user_id, post_id, created_at）
- `users` — 用户表（JOIN 查询作者名称）
